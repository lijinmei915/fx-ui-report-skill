# R04 - 个人月报

## 目标

个人月报用于回答：本月目标完成得怎么样、哪些客户和商机贡献最大、哪些问题反复出现、下月应该押注哪里。

月报不是四份周报拼接；必须输出月度趋势、结构问题和下月重点。

## 输入要求

优先读取或要求用户提供：

- 月度目标和实际完成
- 成交客户、成交金额、回款
- 月初/月末 Pipeline 及变化原因
- 新增客户、新增商机、丢单、停滞商机
- 本月重点跟进记录
- 本月周报或历史记录

## 输出结构

普通月报必须直接渲染为正文，不要放进 `markdown`、`text` 或其他代码块。默认模块顺序：

1. `# [销售姓名] 个人月报 - [月份]`
2. `## 🧭 摘要`：本月最终结果、本月最大问题、下月最重要策略。
3. `## 📊 月度目标完成`：可用表格展示指标、目标、实际、完成率和说明。
4. `## ✅ 本月关键成果`：每条写成“成果一句话 + 证据 + 价值”。
5. `## 💰 成交与 Pipeline`：写成交/回款、Pipeline 变化和结构判断。
6. `## 🤝 重点客户复盘`：写客户/商机、本月变化、关键卡点、当前判断和下月动作。
7. `## ⚠️ 主要风险与损失`：写影响、根因和修正动作。
8. `## 🎯 下月行动计划`：分为 `🔴 P0 必须完成`、`🟡 P1 应该推进`、`🔵 P2 有资源再做`。
9. `## 🧾 数据说明`：人员/部门和 CRM 数据覆盖；使用中文对象名，不展示对象 ApiName、记录 ID 或长串主键；正文必须使用引用块 `>` 弱化展示。
10. `## 💾 CRM 日志保存确认`：按 `R09_journal_storage_confirmation.md` 用中文询问是否保存到 CRM 日志。

## 月度分析规则

- 目标完成率必须结合成交、Pipeline 质量、商机阶段结构一起判断。
- 缺少目标数据时不计算目标完成率，只输出成交、回款、Pipeline 等事实指标。
- Pipeline 增长但深度商机减少，要标注“增长质量存疑”。
- 月内多次出现同类延期或无结论沟通，要归纳为模式问题。
- 丢单或项目失效必须写清楚原因；原因未知时标注“需复盘”。
- 下月行动必须聚焦少数关键客户和关键动作。
- 普通月报直接输出可渲染正文，不要整体包进代码块。
- 除月度目标完成外，关键成果、成交与 Pipeline、重点客户复盘和风险默认用短段落或项目符号，不用表格。
- 数据说明放在最后，只展示人员/部门和 CRM 数据覆盖；CRM 数据使用中文对象名，不展示对象 ApiName、记录 ID 或长串主键；正文必须使用引用块 `>` 弱化展示；不默认展示数据缺口表。
- 用户要求 HTML、网页版、手机查看或自适应页面时，先完成月报正文，再执行 `R10_html_report_output.md`，最后执行日志保存确认。
- 输出月报后必须用中文询问是否保存到 CRM 日志；用户确认前不得写入。

## 输出粒度

### 简洁版（默认）

适合快速查看月度结果：

- 摘要 180 字以内。
- 月度目标完成和成交/Pipeline 保留核心指标。
- 本月关键成果 3-5 条。
- 重点客户复盘最多展开 5 个。
- 主要风险只列中高风险和影响下月 P0 的问题。
- 下月行动 P0 不超过 3 条，P1 不超过 5 条，P2 最多 3 条。

### 详细版

适合月度复盘和策略调整：

- 展开目标、成交、回款、Pipeline 变化原因。
- 重点客户复盘最多展开 10 个。
- 风险和损失写清影响、根因、修正动作和验证标准。
- 可归纳本月重复出现的执行问题或客户质量问题。

无论简洁版还是详细版，都必须保留数据说明和 CRM 日志保存确认。

## 趋势判断

月报至少覆盖以下一种趋势判断：

- 目标完成趋势：变好、持平、变差
- Pipeline 质量趋势：更深、更浅、虚高
- 客户推进趋势：更接近成交、停滞、分散
- 执行质量趋势：承诺兑现变好或变差

## 禁止行为

- 不逐条复制日报或周报。
- 不只写“本月拜访了多少客户”。
- 不把新增浅层商机直接当作健康增长。
- 不输出团队平均、团队排名或主管资源分配建议。
