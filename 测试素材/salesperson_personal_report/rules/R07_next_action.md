# R07 - 下一步行动建议

## 目标

把日报、周报、月报中的结果、待办对账和风险判断，转成销售人员下一步可执行动作。

行动建议不是愿望清单，必须具体到客户、联系人、动作、目标和验证标准。

## 优先级

### P0：必须做

满足任一条件：
- 关键成交节点临近
- 高风险商机需要立即处理
- 客户关键人要求明确交付物
- 逾期事项影响后续商机推进
- 竞品进入关键阶段
- 影响成交、阶段推进、客户关键承诺兑现或重大风险止损
- 需要确认 EB、预算、采购流程、POC 范围、合同/法务节点等关键成交条件

### P1：应该做

满足任一条件：
- 可推动商机进入下一阶段
- 需要维护 Champion 或关键联系人
- 是 P0 的前置依赖
- 能补齐决策链、预算、采购流程等关键缺口

### P2：有资源再做

满足任一条件：
- 新客户初步开发
- 非关键客户关系维护
- 案例、资料、背景信息补充
- 长周期机会培育

## 输出要求

每条 P0/P1 建议必须包含：

- 客户/商机
- 联系人或对象
- 具体动作
- 目标
- 验证标准
- 截止时间
- 结果类型：成交推进 / 阶段推进 / 风险止损 / 承诺兑现 / 关键人确认

示例结构直接作为报表正文输出，不要放进代码块：

- `### 🔴 P0 必须完成`
- [客户A] 联系 CFO 吴总确认预算审批时间
  - 目标：拿到明确审批节点
  - 验证标准：客户给出具体会议或审批日期
  - 截止：本周三
  - 结果类型：关键人确认 / 阶段推进
- `### 🟡 P1 应该推进`
- [客户B] 补齐方案评审材料
  - 目标：推动客户进入报价评审
  - 验证标准：客户确认评审会时间
  - 截止：本周五
- `### 🔵 P2 有资源再做`
- [客户C] 做一次关系维护触达
  - 目标：保持客户活跃
  - 验证标准：获得明确反馈或下一次沟通时间

## 周期差异

| 报告类型 | 行动时间尺度 |
|----------|--------------|
| 日报 | 明日行动，最多 1-2 天内完成 |
| 周报 | 下周行动，通常 3-5 个工作日内完成 |
| 月报 | 下月重点，允许拆成阶段性动作 |

## 数量限制

- P0 不超过 3 条
- P1 不超过 5 条
- P2 不超过 3 条

如果超过上限，按商机金额、成交临近度、风险等级和客户战略价值排序，只保留最重要的动作。

## P0 判定补充

以下事项不能单独作为 P0，除非它直接绑定成交节点、阶段推进或风险止损：

- 同步数据
- 整理记录
- 更新报表
- 普通资料准备
- 内部例行沟通

如果这些事项是关键动作的一部分，应写成具体结果型动作，例如“完成 POC 范围确认并取得 CTO 书面回复”，而不是“同步 POC 信息”。

## 禁止行为

- 不写“继续跟进客户A”这种空泛建议。
- 不把所有事项都放进 P0。
- 不给没有风险或没有推进价值的动作过高优先级。
- 不输出团队资源协调、主管排班或 AE 分工建议。
- 不把低杠杆内部事项包装成 P0。
