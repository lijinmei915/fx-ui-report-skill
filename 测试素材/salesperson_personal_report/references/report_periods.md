# 报告周期与时间口径

本文件定义个人日报、个人周报、个人月报的时间范围和指标口径。生成报表前先确定周期，避免同一指标使用不同时间字段。

## 默认时区

默认使用 `Asia/Shanghai`。

如果用户指定其他时区，以用户指定为准；否则所有“今天、本周、本月”等相对时间都按 `Asia/Shanghai` 解释。

## 日期解析总规则

生成个人日报、周报、月报时，必须先把用户表达解析成明确的 `period_start`、`period_end` 和 `journal_time`，再执行 CRM 查询、报表标题和日志保存预览。三处必须使用同一组日期，不要分别推算。

相对日期必须基于当前运行环境的 `current_date/currentDateTime` 和 `Asia/Shanghai` 解析；不要使用模型训练时间、美国时区或上一轮对话中的日期。用户明确指定日期、周次或月份时，以用户输入为准。

常见表达必须按以下规则解析。所有相对表达都以当前运行环境的 `current_date/currentDateTime` 为基准，不受今天是周几影响：

- `今天`、`今日`：`period_start = period_end = current_date`。
- `昨天`、`昨日`：`period_start = period_end = current_date - 1 天`。
- `某一日`、`YYYY-MM-DD`、`YYYY/M/D`、`YYYY年M月D日`、`MMDD`、`M月D日`：解析成该自然日，`period_start = period_end = 指定日期`；缺少年份时默认使用当前年份，跨年意图明显时先说明或追问。
- `本周`：当前日期所在 ISO 自然周，周一到周日；不能是从当前日期开始或结束的滚动 7 天。
- `上周`：当前 ISO 自然周的前一周，周一到周日；不能包含当前日期，也不能是最近 7 天。
- `某一周`、`YYYY-WXX`、`YYYY年第XX周`、`WXX`、`第XX周`：按 ISO week-year 解析成该周周一到周日；`WXX` 或 `第XX周` 缺少年份时默认使用当前 ISO week-year。
- `某个具体日期所在周`、`5月20日那周`、`2026-05-20所在周`：先解析该具体日期，再取该日期所在 ISO 自然周的周一到周日。
- `本月`：当前日期所在自然月，月初到月末。
- `上月`：当前自然月的前一自然月，月初到月末。
- `某一月`、`YYYY-MM`、`YYYY年M月`、`M月`：解析成该自然月，月初到月末；缺少年份时默认使用当前年份。

日志保存的 `journal_time` 必须使用 `yyyyMMdd`：

- 日报：使用日报对应日期。
- 周报：使用该 ISO 自然周的周日，也就是 `period_end`。
- 月报：使用该自然月最后一天，也就是 `period_end`。

保存预览中展示的“时间”可以是业务友好格式，例如 `<period_start> 至 <period_end>`；内部 payload 的 `journal_time` 必须是 `<period_end_yyyyMMdd>`，不能带横线、时间或范围文本。

## 个人日报

| 项目 | 默认口径 |
|------|----------|
| 时间范围 | 当天 00:00:00 到 23:59:59 |
| 默认日期 | 今天 |
| 日志保存日期 | 日报对应日期，`journal_time = 当天 yyyyMMdd` |
| 跟进记录 | 使用 `ActiveRecordObj.create_time` |
| 日程 | 使用 `ScheduleObj.begin_time` 和 `ScheduleObj.end_time` |
| 外勤 | 使用 `CheckinsObj.new_stat_date` |
| 成交订单 | 使用 `SalesOrderObj.create_time` |
| 回款 | 优先使用 `PaymentObj.payment_time`，没有时使用 `PaymentObj.create_time` |
| 新增客户/线索/商机 | 使用对应对象的 `create_time` |

日报重点看当天事实、当天承诺兑现和明日行动，不做长期能力判断。

## 个人周报

| 项目 | 默认口径 |
|------|----------|
| 时间范围 | 周一 00:00:00 到周日 23:59:59 |
| 默认周 | 本周 |
| 周次格式 | `YYYY-WXX` |
| 日志保存日期 | 该 ISO 自然周周日，`journal_time = period_end yyyyMMdd` |
| 对比周期 | 上一自然周 |
| 周成交 | `SalesOrderObj.create_time` 在本周 |
| 周回款 | `PaymentObj.payment_time` 在本周 |
| 周跟进 | `ActiveRecordObj.create_time` 在本周 |
| 周新增客户/线索/商机 | 对应对象 `create_time` 在本周 |
| Pipeline 快照 | 优先取周期末状态；如只能查询当前状态，必须标注“当前快照” |

如果企业使用非自然周的销售周，以用户或企业规范指定为准，并在报表标题说明实际日期范围。

### 周报日期解析规则

周报默认使用 ISO 自然周：

- 周一是一周第一天，周日是一周最后一天。
- `本周` 按当前日期所在的 ISO 周计算，不从当前日期或周二开始滚动 7 天。
- `上周` 是当前 ISO 周的前一周，仍为周一到周日；`上周 period_end = 当前 ISO 周周一 - 1 天`，因此上周绝不能包含当前日期。
- 周次 `YYYY-WXX` 使用 ISO week-year；跨年周以 ISO 规则为准，不直接用自然年 1 月 1 日所在年份推断。
- 生成标题、正文开头、CRM 查询时间过滤和日志保存时间必须使用同一个周起止日期。
- 输出前必须校验周次与日期范围一致；如果标题写 `<YYYY-WXX>`，日期范围必须覆盖该 ISO 周的周一到周日。
- 周报写入日志时，`journal_time` 必须使用该周周日，即 `period_end` 的 `yyyyMMdd`，不是当前生成日，也不是规则文档中的示例日期。

计算校验：

```text
week_start = 当前日期所在周的周一
week_end = week_start + 6 天
previous_week_start = week_start - 7 天
previous_week_end = week_start - 1 天

specified_date_week_start = 指定日期所在 ISO 周的周一
specified_date_week_end = specified_date_week_start + 6 天
```

校验示例：如果当前日期是某个周四，`本周` 必须解析为该周四所在的 ISO 自然周，日期范围从当周周一到当周周日；不要写成从周四、昨天、周二或其他非周一起点滚动 7 天。

回归校验：当 `current_date = 2026-05-25`（周一，Asia/Shanghai）时，`本周` 必须是 `2026-05-25 至 2026-05-31`，`上周` 必须是 `2026-05-18 至 2026-05-24`，上周 `journal_time` 必须是 `20260524`；绝不能把 `上周` 写成 `2026-05-19 至 2026-05-25`。本样例只用于防止滚动 7 天错误，不得在其他日期当作当前日期。

具体周次校验：如果用户指定 `2026-W21`，必须解析为 `2026-05-18 至 2026-05-24`，`journal_time = 20260524`。如果用户说 `2026-05-20 那周`，也必须解析为 `2026-05-18 至 2026-05-24`。不要把指定日期当天当作周报起点。

## 个人月报

| 项目 | 默认口径 |
|------|----------|
| 时间范围 | 月初 00:00:00 到月末 23:59:59 |
| 默认月 | 本月 |
| 月份格式 | `YYYY-MM` |
| 日志保存日期 | 该自然月最后一天，`journal_time = period_end yyyyMMdd` |
| 对比周期 | 上一自然月 |
| 月成交 | `SalesOrderObj.create_time` 在本月 |
| 月回款 | `PaymentObj.payment_time` 在本月 |
| 月新增客户/线索/商机 | 对应对象 `create_time` 在本月 |
| 月末 Pipeline | 优先取月末快照；如只能查询当前状态，必须标注“当前快照” |

月报重点看目标完成、Pipeline 结构、客户趋势、损失原因和下月重点，不逐周复制周报。

## 目标完成率口径

目标完成率 = 实际完成 / 目标。

实际完成可按业务要求选择成交金额或回款金额：

- 默认使用成交金额。
- 如果用户明确问回款完成率，使用回款金额。
- 如果个人考核以回款为准，使用回款金额并说明口径。

目标来源优先级：

1. CRM 中可查询到的目标/配额数据。
2. 用户提供的个人目标。
3. `JournalObj` 历史日志中有明确来源的目标。
4. 本地 `memory/` 中有明确来源的目标，仅在 CRM/JournalObj 不可用时兜底参考。

如果没有目标数据，不计算目标完成率；只输出成交、回款、Pipeline 等事实指标，并标注“目标数据缺失”。

## Pipeline 口径

Pipeline 默认包含进行中的商机，不包含赢单、输单、无效商机。

如用户要求包含历史赢单或全部商机，必须在报表中说明口径。

Pipeline 对比优先使用周期起止快照；如果没有历史快照，只能使用当前 CRM 状态时，必须标注：

```text
Pipeline 为当前 CRM 快照，无法严格还原周期内变化。
```

## 数据冲突处理

- CRM 时间字段与用户自述冲突：优先采用 CRM 字段，并标注冲突。
- 成交时间与回款时间不同：分别统计成交和回款，不混算。
- 日程时间与销售记录时间不同：分别作为计划动作和实际跟进。
- 周期边界不确定：先追问；若用户要求直接输出，则标注“时间范围待确认”。
