# 销售管理者团队报表 Memory 模板

本模板仅用于 CRM 或 CRM 日志不可用时的本地兜底归档。真实运行优先使用 CRM 标准对象、自定义对象和 CRM 日志，不要把本地 memory 作为默认事实源或默认归档源。

默认不自动更新本地 memory。只有用户明确要求本地归档、CRM/日志不可用，或测试场景需要时才参考以下模板。

## 记忆架构

```text
memory/
├── TEMPLATE.md
├── daily/
│   └── YYYY-MM-DD.md
├── weekly/
│   └── YYYY-WXX.md
├── monthly/
│   └── YYYY-MM.md
├── ae_health/
│   └── [AE姓名].md
├── cross_risk/
│   └── [风险ID].md
└── management_action/
    └── [周期].md
```

## 使用规则

- CRM/CRM 日志与 AE 自述冲突：保留两者，标注“待核实”，优先采用 CRM 事实。
- CRM/CRM 日志与历史 memory 冲突：优先采用 CRM/CRM 日志，记录冲突点和来源。
- 团队 memory 与个人报表 memory 冲突：记录冲突点和来源，不直接覆盖。
- 缺少 AE 数据：只在受影响模块说明影响，不默认表现正常。
- 未获得写入权限：只在输出中说明“建议写入本地 memory 的内容”，不要声称已经写入。
- 团队 memory 记录管理视角，不复制个人日报、周报、月报全文。
- AE 健康度用于识别趋势，不应用单期异常直接定性能力。
- 跨 AE 风险必须记录涉及范围和证据。
- 管理动作必须记录验证标准，否则无法追踪是否完成。
- 面向用户输出时使用中文对象名，不展示对象 ApiName、记录 ID 或长串主键。
