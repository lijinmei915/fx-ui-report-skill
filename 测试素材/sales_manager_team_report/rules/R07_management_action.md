# R07 - 管理行动建议

## 目标

把团队数据聚合和跨 AE 风险转成销售管理者可执行的管理动作。

行动建议必须具体到负责人、对象、动作、截止时间和验证标准。

生成新管理动作前，优先检查上一周期 CRM 日志、任务、日程或销售记录中的管理动作；CRM/日志不可用时才参考 `memory/management_action/`。未完成的 P0/P1 不允许无声消失，必须说明状态、影响和本期处理方式。

## 优先级

### P0：主管必须亲自处理

满足任一条件：
- 高风险关键商机需要主管联系客户
- 竞品集中渗透影响多个 AE 或重点行业
- 某 AE 连续多期异常且影响团队目标
- 主管上期 P0 未完成并继续影响结果
- 重大客户或大额商机进入关键决策节点

### P1：团队协调处理

满足任一条件：
- 需要跨 AE 协作或资源调配
- 需要统一话术、案例、行业打法
- 需要净化 Pipeline 或修正数据口径
- 需要对某 AE 做辅导、陪访、复盘

### P2：常规管理动作

满足任一条件：
- 常规客户经营
- 信息补充
- 低风险商机培育
- 非紧急经验沉淀

## 输出要求

每条 P0/P1 必须包含：

- 负责人
- 涉及 AE / 客户 / 商机
- 具体动作
- 目标
- 截止时间
- 验证标准

## 管理动作闭环

输出本期行动前，先给出上一期动作闭环：

上期管理动作闭环直接作为报表正文模块输出，不要放进代码块。可用表格展示：动作、负责人、上期状态、验证证据、本期处理。

处理规则：

- 已完成：写明结果和证据。
- 未完成但仍重要：延续到本期 P0/P1，并说明影响。
- 未完成但已失效：说明失效原因，不再延续。
- 状态未知：在管理动作闭环中说明缺少验证证据，必要时进入 P1 补确认。

示例结构：

管理动作直接作为报表正文输出，不要放进代码块：

- `### 🔴 P0 主管必须亲自处理`
- [华科集团] 主管介入
  - 负责人：销售总监 + 张明
  - 动作：联系客户 VP 确认真实卡点
  - 目标：拿到明确推进时间或终止原因
  - 截止：本周三
  - 验证标准：客户 VP 给出明确结论

## 资源协调类型

| 类型 | 说明 |
|------|------|
| 主管介入 | 主管直接联系客户、参加会议、推进关键节点 |
| AE 协作 | 一个 AE 支援另一个 AE 的行业经验、关系或方案 |
| 团队统一动作 | 周会统一话术、竞品应对、Pipeline 净化 |
| 经验复制 | 把有效打法沉淀给全团队 |
| 资源调整 | 商机分配、陪访安排、重点客户优先级调整 |

## 数量限制

- P0 不超过 3 条
- P1 不超过 5 条
- P2 不超过 3 条

如果超过上限，按影响目标金额、风险等级、成交临近度、涉及 AE 数量排序。

## 禁止行为

- 不写“加强管理、继续关注、提高执行力”这类空泛建议。
- 不把所有事项都放入 P0。
- 不给没有负责人或验证标准的建议。
- 不让“全体 AE”负责含糊任务；如需全体参与，仍要指定牵头负责人。
- 不把个人行动建议替代管理动作。
