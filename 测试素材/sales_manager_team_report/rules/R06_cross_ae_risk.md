# R06 - 跨 AE 风险分析

## 目标

识别只有从团队视角才能发现的系统性风险。不要重复个人报表中的单点风险，除非它已经影响团队判断或资源分配。

## 风险类型

### 1. 竞品集中渗透

触发条件：
- 同一行业或相似客户中，2 个以上 AE 报告竞品信号
- 多个客户同时进入“内部讨论、比价、重新评估”
- 竞品在团队目标行业近期出现强信号

输出必须包含：竞品、涉及客户、涉及 AE、证据、影响金额、建议动作。

### 2. Pipeline 质量下降

触发条件：
- Pipeline 增长但成交率下降
- 新增商机多数处于浅层阶段
- 30% 以上 Pipeline 超过 4 周无实质推进
- 深度阶段商机占比下降

不要只看金额增长；必须判断质量。

### 3. AE 表现或状态异常

触发条件：
- 某 AE 连续多期低于团队平均
- 逾期、延期或高风险商机持续增加
- Pipeline 持续下降
- 数据缺失、跟进记录缺失影响管理判断

注意：不要轻易定性为“能力差”。优先区分客户结构、资源不足、执行问题、数据缺失和市场原因。

### 4. 资源过度集中

触发条件：
- 某 AE 成交或 Pipeline 占团队过高
- 某客户占某 AE Pipeline 过高
- 某行业占团队 Pipeline 过高
- 主管介入事项集中在少数人或少数客户

### 5. 团队执行风险

触发条件：
- 团队承诺兑现率持续下降
- 多个 AE 出现相似延期理由
- P0 事项跨期未完成
- 主管上期行动没有验证结果

## 输出格式

跨 AE 风险直接作为报表正文模块输出，不要放进代码块。可用表格展示：风险、涉及 AE/客户、等级、依据、管理动作。

等级使用：高、中、低、待确认。

依据必须是可追溯信息，如：

- 涉及 3 个 AE / 5 个客户
- Pipeline 增长 20% 但深度商机下降
- 某 AE 连续 3 周低于团队平均 20%
- 同一竞品在两个行业组出现

## 风险升级

- 同一风险连续 2 期出现：升级为需关注
- 同一风险连续 3 期出现：升级为高风险
- 高风险影响当期目标或关键客户：进入 P0 管理动作

## 禁止行为

- 不重复罗列所有个人风险。
- 不输出没有涉及 AE/客户/金额/证据的空泛风险。
- 不把短期波动直接定性为长期能力问题。
- 不把数据缺失当成表现正常。
- 风险数量默认控制在 3 个以内，除非用户要求完整风险清单。
