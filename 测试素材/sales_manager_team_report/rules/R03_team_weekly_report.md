# R03 - 团队周报

## 目标

团队周报用于回答：本周团队目标完成如何、哪些 AE 和商机异常、跨 AE 风险是什么、下周主管应该抓什么。

团队周报不是个人周报拼接，而是管理视角的汇总和判断。

## 执行顺序

1. 执行 `R05_team_aggregation.md` 汇总团队数据。
2. 执行 `R06_cross_ae_risk.md` 识别跨 AE 风险。
3. 执行 `R07_management_action.md` 生成下周管理动作。
4. 用户要求 HTML 输出时，执行 `R10_html_report_output.md` 生成自适应 HTML。
5. 执行 `R09_team_journal_storage_confirmation.md` 询问是否保存到 CRM 日志。

## 输入要求

优先读取或要求用户提供：

- 本周团队目标、成交、回款、Pipeline、任务、风险数据
- 各 AE 的 CRM 明细、日志或周报归档
- 上周团队日志、团队周报或本地历史记录
- 本周重点客户、行业和竞品信号

## 输出结构

普通周报必须直接渲染为正文，不要放进 `markdown`、`text` 或其他代码块。默认模块顺序：

1. `# 团队周报 - [团队/范围] - [周次/日期范围]`
2. `## 🧭 摘要`：团队整体状态、本周最大风险、下周主管最重要动作。
3. `## 📊 团队关键指标`：可用表格展示指标、本周、上周、变化和判断。
4. `## ✅ 本周关键结果`：每条写成“结果一句话 + 证据 + 管理意义”。
5. `## 👥 AE 表现与异常`：可用表格展示 AE、目标/结果、Pipeline、风险/异常和管理判断。
6. `## 🤝 重点客户与商机`：写客户/商机、本周合并进展、负责 AE、当前判断和主管动作。
7. `## ⚠️ 跨 AE 风险`：写涉及 AE/客户/行业、依据和管理动作。
8. `## 🎯 下周团队行动计划`：分为 `🔴 P0 主管必须亲自处理`、`🟡 P1 团队协调处理`、`🔵 P2 常规跟进`。
9. `## 🧾 数据说明`：团队/AE 覆盖和 CRM 数据覆盖；使用中文对象名，不展示对象 ApiName、记录 ID 或长串主键；正文必须使用引用块 `>` 弱化展示。
10. `## 💾 CRM 日志保存确认`：按 `R09_team_journal_storage_confirmation.md` 用中文询问是否保存到 CRM 日志。

## 周报重点

- 必须给出团队整体状态：良好、关注、危险或待确认。
- 必须解释 Pipeline、成交、回款、目标完成率的主要变化。
- 必须识别表现异常 AE，但不要在证据不足时做能力定性。
- 必须把跨 AE 风险和个人客户风险区分开。
- 必须给出主管下周最该处理的 2-3 件事。
- 必须说明本周团队成员覆盖情况；无法确认总人数时，不输出覆盖百分比。
- 普通周报直接输出可渲染正文，不要整体包进代码块。
- 数据说明放在最后，只展示团队/AE 覆盖和 CRM 数据覆盖；正文必须使用引用块 `>` 弱化展示；CRM 数据使用中文对象名，不展示对象 ApiName、记录 ID 或长串主键；不默认展示数据缺口表。
- 用户要求 HTML 输出时，先完成周报内容，再执行 `R10_html_report_output.md` 生成自适应 HTML。
- 输出周报后必须用中文询问是否保存到 CRM 日志；用户确认前不得写入。

## 输出粒度

### 简洁版（默认）

适合周会快速浏览和向上汇报：

- 摘要 150 字以内，必须包含团队状态、关键数字、最大风险和下周主管动作。
- 团队关键指标保留 6-10 个核心指标。
- 本周关键结果 3-5 条。
- AE 表现与异常只展开关键 AE、异常 AE 和数据缺失 AE。
- 重点客户与商机最多展开 5 个。
- 跨 AE 风险只列中高风险或影响团队目标的系统性风险。
- 下周团队行动 P0 不超过 3 条，P1 不超过 5 条，P2 最多 3 条。

### 详细版

适合团队复盘、周会材料或管理诊断：

- 团队关键指标可补充口径解释、变化原因和环比判断。
- AE 表现可展开到全部已覆盖 AE，包含目标/结果、Pipeline、风险、管理判断。
- 重点客户与商机最多展开 10 个，写负责 AE、阶段变化、金额、风险和主管动作。
- 跨 AE 风险写清涉及 AE/客户/行业、依据、影响和管理动作。
- 管理动作闭环逐条展开上周 P0/P1 的完成情况、证据、延期原因和新动作。

无论简洁版还是详细版，都必须保留数据说明和 CRM 日志保存确认。详细版不得把多个个人周报简单拼接成团队周报。

## 数据缺失处理

如果某个 AE 缺少数据，不单独输出“数据缺口与口径”表；只在受影响模块内说明：

[AE姓名] 缺少 [日报/周报/CRM字段]，因此团队汇总中该 AE 的 [指标] 无法判断。

缺失 AE 不应被默认为表现正常，也不应被直接判定异常。
