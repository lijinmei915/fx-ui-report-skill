# R05 - 团队数据聚合

## 目标

把 CRM 数据、CRM 日志、用户输入和必要的本地兜底资料聚合成团队视图，用于团队日报、周报和月报。

聚合不是简单相加；必须识别异常、结构和变化原因。

## 数据来源

优先读取：

- CRM 客户、商机、跟进、成交、回款数据；任务数据如存在则读取
- CRM 日志对象中的团队历史日报、周报、月报和上期管理动作
- CRM 自定义对象中用户选择后的团队事实数据源
- 用户补充说明
- 本 skill 的 `memory/`，仅在 CRM/日志不可用时兜底参考
- 个人销售报表 memory，仅在 CRM/日志不可用时作为 AE 自述兜底参考

## 聚合维度

| 维度 | 聚合内容 |
|------|----------|
| 团队整体 | 目标完成、成交、回款、Pipeline、风险数量 |
| AE 维度 | 每个 AE 的目标完成、Pipeline、逾期、风险、成交 |
| 客户/商机维度 | 高金额、高风险、临近成交、停滞商机 |
| 阶段维度 | Pipeline 阶段分布、深度商机占比 |
| 时间维度 | 本期 vs 上期、月内趋势、连续异常 |

## 覆盖率口径

聚合前先执行 `R08_team_scope_and_coverage.md`。

团队指标必须说明分母：

- 已确认 AE 总数明确时，可输出“已覆盖 N/M”。
- AE 总数不明确时，只能输出“覆盖范围待确认”，不要输出百分比。
- 某 AE 部分字段缺失时，只对可确认字段参与统计；缺失字段不按 0 处理。
- 某 AE 完全缺失时，不纳入金额类事实汇总，但必须在受影响模块内说明影响。

示例：

本周 CRM 数据覆盖 5/7 名已确认 AE；2 名 AE 缺少本周跟进和 Pipeline 数据，因此团队 Pipeline 与风险判断可能低估。

## AE 横向对比

输出时至少包含：

AE 横向对比可直接用表格输出，不要放进代码块。字段至少包含：AE、目标/结果、Pipeline、成交/回款、风险/异常、管理判断。

异常标注参考：

- 目标完成率低于团队平均 20% 以上
- 连续两期目标完成低于 50%
- Pipeline 单一客户占比超过 40%
- 逾期任务超过 2 个
- 高风险商机连续两期未改善
- 数据缺失影响判断

## Pipeline 汇总

必须说明：

- 总 Pipeline
- 对比上期变化
- 新增、升阶、成交转出、输单/无效
- 阶段分布
- 深度阶段占比
- 是否存在浅层增长或单点集中

如果 Pipeline 变化无法解释，在 Pipeline 模块内说明“变化原因缺少证据”，不要替 AE 补理由。

## 团队健康度

团队健康度至少参考：

- 目标完成
- Pipeline 质量
- 逾期控制
- 风险控制

输出等级：

| 等级 | 说明 |
|------|------|
| 良好 | 关键指标正常，无明显高风险 |
| 关注 | 1-2 项指标异常，需要主管关注 |
| 危险 | 多项指标异常，或高风险集中出现 |
| 待确认 | 数据缺失严重，无法可靠判断 |

## 缺失处理

不要单独输出“数据缺口”表。缺失影响判断时，在对应模块内说明：

因 [AE/对象/字段] 缺少 [数据]，本模块的 [指标/判断] 可能 [低估/无法判断]。

不要把缺失数据的 AE 排除后直接计算“健康团队”结论。

如果缺失数据影响团队健康度，整体状态优先标为“待确认”或“关注”，不要输出“良好”。
