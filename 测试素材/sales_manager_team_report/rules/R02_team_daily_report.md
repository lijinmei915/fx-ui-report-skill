# R02 - 团队日报

## 目标

团队日报用于回答：今天团队发生了什么关键变化、哪些异常需要主管当天处理、明天团队要优先做什么。

日报强调即时管理动作，不做长篇复盘。

## 输入要求

优先读取或要求用户提供：

- 当日各 AE 跟进记录、日程、外勤、任务完成情况
- 当日成交、回款、重点商机阶段变化
- 当日新增高风险客户或关键客户异常
- 当日未完成的 P0 任务
- 明日关键日程和重点动作

## 输出结构

普通日报必须直接渲染为正文，不要放进 `markdown`、`text` 或其他代码块。默认模块顺序：

1. `# 团队日报 - [团队/范围] - [日期]`
2. `## 🧭 摘要`：今日团队最重要结果、今日最大异常、明日主管最重要动作。
3. `## 📊 今日关键指标`：可用表格展示指标、今日和判断。
4. `## ✅ 今日关键结果`：每条写成“结果一句话 + 证据 + 管理意义”。
5. `## 👥 AE 今日表现与异常`：可用表格展示 AE、关键结果、未完成事项、风险/异常和管理判断。
6. `## 🤝 重点客户与商机`：写客户/商机、今日变化、负责 AE、当前判断和明日动作。
7. `## ⚠️ 今日需主管关注`：写 AE/客户/商机、异常判断、依据和主管动作。
8. `## 🎯 明日团队行动`：分为 `🔴 P0 主管必须亲自处理`、`🟡 P1 团队协调处理`、`🔵 P2 常规跟进`。
9. `## 🧾 数据说明`：团队/AE 覆盖和 CRM 数据覆盖；使用中文对象名，不展示对象 ApiName、记录 ID 或长串主键；正文必须使用引用块 `>` 弱化展示。
10. `## 💾 CRM 日志保存确认`：按 `R09_team_journal_storage_confirmation.md` 用中文询问是否保存到 CRM 日志。

## 日报判断重点

- 当日成交、回款、阶段升阶优先写。
- 高风险客户、关键 AE 异常、重要 P0 未完成必须写。
- 只写影响管理动作的信息，普通流水账合并成一句“其他正常推进”。
- 明日 P0 不超过 3 条。
- 必须写清本日报覆盖了哪些 AE；未提交或查不到数据的 AE 只在影响结论的模块内说明。
- 普通日报直接输出可渲染正文，不要整体包进代码块。
- 数据说明放在最后，只展示团队/AE 覆盖和 CRM 数据覆盖；正文必须使用引用块 `>` 弱化展示；CRM 数据使用中文对象名，不展示对象 ApiName、记录 ID 或长串主键；不默认展示数据缺口表。
- 用户要求 HTML 输出时，先完成日报内容，再执行 `R10_html_report_output.md` 生成自适应 HTML。
- 输出日报后必须用中文询问是否保存到 CRM 日志；用户确认前不得写入。

## 输出粒度

### 简洁版（默认）

适合主管当天快速判断：

- 摘要 100 字以内。
- 今日关键指标保留 5-8 个核心指标。
- 今日关键结果 3-5 条。
- AE 今日表现只列关键结果、异常 AE 和未提交/待确认 AE。
- 重点客户与商机最多展开 3-5 个。
- 今日需主管关注只列高风险或影响明日 P0 的异常。
- 明日团队行动 P0 不超过 3 条，P1 不超过 3 条，P2 可省略或最多 2 条。

### 详细版

适合日会复盘或追踪执行明细：

- AE 今日表现可展开到全部已覆盖 AE。
- 重点客户与商机最多展开 10 个。
- 异常写完整依据、影响、建议主管动作和截止时间。
- 可补充普通客户/商机概览，但不要拼接个人流水账。

无论简洁版还是详细版，都必须保留数据说明和 CRM 日志保存确认。

## 异常识别

日报重点关注：

- 关键客户当天未回复或取消会议
- 当日 P0 未完成且影响成交节点
- 重要商机阶段倒退或结单日期变化
- AE 当日无有效跟进且负责关键商机
- 新增竞品、价格、法务、采购阻力信号

## 禁止行为

- 不把所有 AE 的跟进流水直接拼接。
- 不输出没有管理动作价值的细节。
- 不把个人日报改写成团队日报。
- 不在数据缺失时假装全员已提交。
