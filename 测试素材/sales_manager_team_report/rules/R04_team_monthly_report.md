# R04 - 团队月报

## 目标

团队月报用于回答：本月团队经营结果如何、Pipeline 和成交结构是否健康、哪些 AE/行业/客户形成趋势性问题、下月管理重点是什么。

月报不是周报堆叠；必须输出趋势、结构和管理判断。

## 输入要求

优先读取或要求用户提供：

- 本月团队目标和实际完成
- 本月成交、回款、Pipeline 期初/期末和阶段结构
- AE 表现分层、异常和趋势
- 本月重点行业、重点客户、重点商机和竞品信号
- 本月团队日志、团队周报或本地历史记录

## 输出结构

普通月报必须直接渲染为正文，不要放进 `markdown`、`text` 或其他代码块。默认模块顺序：

1. `# 团队月报 - [团队/范围] - [月份]`
2. `## 🧭 摘要`：本月团队最终结果、本月最大管理问题、下月最重要管理策略。
3. `## 📊 月度经营指标`：可用表格展示指标、目标、实际、完成率/变化和判断。
4. `## ✅ 本月关键结果`：每条写成“结果一句话 + 证据 + 管理意义”。
5. `## 👥 AE 表现分层`：可用表格展示分层、AE、依据和管理判断。
6. `## 💰 成交与 Pipeline 结构`：写成交/回款、Pipeline 变化和结构判断。
7. `## 🤝 重点客户/行业复盘`：写客户/行业/商机、本月变化、负责 AE、关键卡点和下月管理动作。
8. `## ⚠️ 主要风险与管理问题`：写影响、根因和修正动作。
9. `## 🎯 下月团队行动计划`：分为 `🔴 P0 主管必须亲自处理`、`🟡 P1 团队协调处理`、`🔵 P2 常规跟进`。
10. `## 🧾 数据说明`：团队/AE 覆盖和 CRM 数据覆盖；使用中文对象名，不展示对象 ApiName、记录 ID 或长串主键；正文必须使用引用块 `>` 弱化展示。
11. `## 💾 CRM 日志保存确认`：按 `R09_team_journal_storage_confirmation.md` 用中文询问是否保存到 CRM 日志。

## 月报分析规则

- 目标完成率必须结合成交、回款、Pipeline 质量和 AE 结构一起判断。
- 缺少目标数据时不计算目标完成率，只输出成交、回款、Pipeline 等事实指标。
- Pipeline 增长但深度商机减少，要标注“增长质量存疑”。
- 同类风险连续出现，要归纳为团队能力、管理机制或市场结构问题。
- 下月 P0 应聚焦主管必须抓的关键客户、关键 AE 或关键机制。
- 月度趋势必须说明成员范围是否一致；如果本月和上月 AE 范围不同，在相关趋势模块内说明口径变化。
- 普通月报直接输出可渲染正文，不要整体包进代码块。
- 数据说明放在最后，只展示团队/AE 覆盖和 CRM 数据覆盖；正文必须使用引用块 `>` 弱化展示；CRM 数据使用中文对象名，不展示对象 ApiName、记录 ID 或长串主键；不默认展示数据缺口表。
- 用户要求 HTML 输出时，先完成月报内容，再执行 `R10_html_report_output.md` 生成自适应 HTML。
- 输出月报后必须用中文询问是否保存到 CRM 日志；用户确认前不得写入。

## 输出粒度

### 简洁版（默认）

适合管理层快速查看月度经营结果：

- 摘要 180 字以内。
- 月度经营指标保留目标、成交、回款、Pipeline、AE 覆盖等核心指标。
- 本月关键结果 3-5 条。
- AE 表现分层只展示分层结论、关键 AE 和异常 AE。
- 重点客户/行业复盘最多展开 5 个。
- 主要风险只列影响下月 P0 或团队目标的风险。
- 下月团队行动 P0 不超过 3 条，P1 不超过 5 条，P2 最多 3 条。

### 详细版

适合月度经营复盘和管理诊断：

- 展开目标、成交、回款、Pipeline 结构和阶段变化原因。
- AE 表现分层可展开全部已覆盖 AE，并说明分层依据。
- 重点客户/行业复盘最多展开 10 个。
- 主要风险写清影响、根因、涉及 AE/客户/行业、修正动作和验证标准。
- 可归纳团队能力短板、机制问题和下月管理节奏。

无论简洁版还是详细版，都必须保留数据说明和 CRM 日志保存确认。

## 趋势判断

月报至少覆盖以下趋势：

- 团队目标完成趋势
- Pipeline 质量趋势
- AE 表现分层变化
- 客户/行业风险趋势
- 管理动作兑现趋势

## 禁止行为

- 不逐周复制团队周报。
- 不只写“本月团队拜访多少客户”。
- 不用团队平均掩盖关键 AE 或关键客户风险。
- 不输出没有负责人和验证标准的管理动作。
