# 高层陪访简报

## 目标

为 AE、销售经理或销售管理者生成一份能直接转给领导阅读的客户陪访简报。高层陪访场景的核心不是“AE 今天怎么打”，而是帮助领导在 3 分钟内判断：

- 这个客户为什么值得我出面？
- 这次陪访要我扮演什么角色？
- 客户高层是谁，和我们是什么关系？
- 当前最大风险是什么，我可以如何化解？
- 哪些话不能说，哪些承诺不能替 AE 做？
- 会后 AE 应该如何承接领导动作？

默认输出结构化 Markdown。只有用户明确要求 HTML、移动端或可分享版本时，才使用模板资产。

## 适用场景

使用本技能处理：

- `/escalate [客户名称]`
- `/escalate [客户名称] --full`
- “帮我写一份领导陪访简报”
- “我要请 VP/总监/老板陪访这个客户”
- “明天领导要一起见客户，帮我整理给领导看的简报”
- “这个商机要升级到高层，帮我准备汇报材料”

不要用于：

- AE 自用的见客前 1 分钟行动简报。
- 团队交接、客户转手或接手人继承客户上下文。
- 销售周报、团队 review 或客户组合盘点。
- 泛客户360认知档案。
- 完整决策链诊断、EB/Sponsor识别或角色覆盖评分；交给 `decision-chain`。
- 方案汇报故事线、页级大纲或Q&A演练；交给 `proposal-design`。
- 报价、让价或商务谈判策略；交给 `quotation-strategy`。
- 销售经理1:1辅导、过程检查或训练动作；交给 `sales-manager-coaching`。

本技能负责生成高层陪访简报，可以汇总客户事实、商机价值、风险、决策链和领导介入点，但不做深度业务诊断。用户要决策链诊断转 `decision-chain`，要方案汇报故事线转 `proposal-design`，要报价或商务谈判转 `quotation-strategy`，要销售经理辅导转 `sales-manager-coaching`。

## 输出原则

- 领导视角：所有内容都要服务“领导是否该去、去了做什么、不能做什么”。
- 结论先行：先给陪访结论摘要，再展开证据。
- 价值要具体：商机金额、战略意义、行业标杆、续约/增购、竞争窗口要说清楚。
- 风险要诚实：竞品、预算、关系、交付、回款、承诺、客户内部政治不能弱化。
- 角色要明确：不要让领导做 AE 的基础跟进工作；领导动作应是背书、定调、破局、探测、稳关系。
- 边界要写清：哪些承诺不能现场给，哪些敏感问题不要主动提，哪些人不要绕过。
- 不输出 AE 内部语言：不要写“今日目标”“P0/P1/P2”“就绪度”“确认出发”。

## 数据优先级

1. CRM Opportunity：当前商机、阶段、金额、预计结单、预算/审批、赢率。
2. CRM Account：客户基础信息、行业、规模、成交状态、历史合作。
3. Memory：客户高层关系、历史博弈、承诺边界、领导曾介入记录、敏感点。
4. CRM Contacts：客户高层、决策链、Coach、Gatekeeper、技术/业务/采购关键人。
5. CRM Activity：最近互动、客户原话、客户情绪、承诺、参会人。
6. 竞品、合同、订单、回款、CSM 预警、流失原因：`--full` 或高风险客户时使用。
7. References：CLI、CRM 字段、行业基准、竞品、领导陪访打法。
8. 公开信息：当用户要求或陪访价值判断依赖客户新闻、融资、财报、政策动态时使用。

## 处理流程

1. 识别客户。
   - 需要从 CRM 查询时，先读取 `references/cli_reference.md`。
   - 查询客户 `AccountObj`，拿到客户 `_id`。
   - 多个候选客户时，先让用户选择。

2. 明确陪访请求。
   - 识别申请人、拟陪访领导、客户参会高层、陪访时间、陪访形式。
   - 如果用户没有说明领导角色，按“未知”处理，不阻塞生成，但必须写入待确认项。

3. 检查 Memory。
   - 读取 `memory/customers/[客户名称]/` 下已有记忆。
   - 重点查找高层关系、历史升级、领导曾经承诺、客户敏感点、关键博弈事件。
   - 如果 Memory 不足，摘要中必须提示：“Memory 不足，需 AE 补充高层关系和历史敏感点”。
   - 只有创建或更新记忆结构时，读取 `memory/template.md`。

4. 拉取陪访所需 CRM 数据。
   - 默认读取客户、商机、联系人、最近互动、竞品明细。
   - `--full`、已成交/续约/增购/高风险客户，再读取合同、订单、回款、CSM 预警、流失原因。
   - 字段含义不清时，读取 CRM 字段参考。

5. 加载规则。
   - 目标规则集应为：
     - `rules/R01_visit_context.md`
     - `rules/R02_opportunity_value.md`
     - `rules/R03_executive_stakeholder_map.md`
     - `rules/R04_intervention_points.md`
     - `rules/R05_risk_sensitive_points.md`
     - `rules/R06_visit_objectives.md`
     - `rules/R07_output_assembly.md`
     - `rules/R08_financial_health.md`

6. 生成高层陪访简报。
   - 标准模式输出领导必读摘要、陪访背景、商机价值、客户高层与关键人、领导可介入点、风险与敏感点、陪访目标、会后承接建议。
   - `--full` 模式追加历史合作、合同订单、回款、客户健康、续约/增购和流失风险。

## 资源路由

- 需要从 CRM 拉取数据时，读取 `references/cli_reference.md`。
- 需要解释 CRM 对象或字段时，读取 `references/crm_schema.md`。
- 需要判断高层陪访打法、领导角色和介入边界时，读取 `references/executive_visit_reference.md`。
- 需要行业背景、采购周期或商机价值假设时，读取 `references/industry_benchmarks.md`。
- 已知或疑似竞品时，读取 `references/competitive_reference.md`。
- 需要 HTML 输出时，使用 `assets/templates/executive_visit.html`；当前模板为内联响应式样式，不需要额外加载 CSS。桌面端应使用双列阅读区，移动端应自动降为单列，不能输出只适配单一视口的固定宽度页面。
- 不从 `rules/` 加载 HTML 或 CSS；`rules/` 只放高层陪访简报的分析和输出规则。

## 输出契约

```markdown
# 高层陪访简报 — {客户名称}

> 陪访日期：{date}
> 申请人：{AE/销售经理/未知}
> 拟陪访领导：{姓名/职务/未知}
> 客户参会高层：{姓名/职务/未知}
> Memory 状态：完整/部分/不足
> 置信度说明：确认 / 推断 / 未知

## 0. 领导必读摘要
**一句话陪访结论：** {是否值得领导出面 + 最大价值 + 最大风险 + 领导角色}

| 摘要项 | 结论 | 置信度 |
| --- | --- | --- |
| 是否建议陪访 | 建议/谨慎建议/暂不建议/信息不足 | 确认/推断/未知 |
| 领导出面价值 |  |  |
| 当前商机状态 |  |  |
| 最大风险 |  |  |
| 客户关键高层 |  |  |
| 领导本次要做什么 |  |  |
| 领导不能做什么 |  |  |

## 1. 陪访背景
{为什么申请领导陪访、当前客户/商机状态、会议背景}

## 2. 商机价值
{金额、阶段、战略意义、行业标杆、续约/增购、竞争窗口}

## 3. 客户高层与关键人
{客户高层、EB/TB/UX/Coach/Gatekeeper、关系温度、联络注意}

## 4. 领导可介入点
{领导以什么身份介入、解决什么问题、预期效果、边界}

## 5. 风险与敏感点
{竞品、预算、关系、交付、回款、历史承诺、客户内部政治、不能踩的坑}

## 6. 陪访目标
{领导本次会议 1-3 个目标、建议话术方向、需要 AE 承接的动作}

## 7. 会后承接建议
{AE/销售经理/CSM/售前会后怎么接住领导动作}

## 8. 历史合作、财务与客户健康（仅 --full）
{合同、订单、回款、CSM 预警、流失原因、续约/增购信号}
```

## 禁止行为

- Memory 不足时，不提示就生成完整陪访结论。
- 让领导做 AE 的基础跟进工作，如“催客户反馈”“推进预算”。
- 输出 AE 内部语言，如“今日目标”“P0/P1/P2”“就绪度”“确认出发”。
- 只说客户好、不报风险。
- 把行业通用推断写成客户真实痛点。
- 隐瞒竞品、预算、交付、回款、客户投诉或历史承诺风险。
- 输出超过 3 个陪访目标、超过 3 条最高风险、超过 5 个关键人。
