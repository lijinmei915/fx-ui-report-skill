# 高层陪访记忆模板

## 使用说明

本模板用于沉淀“领导陪访前后必须继承”的客户知识。它不是完整会议纪要，也不是 CRM 字段备份；只记录会影响领导是否出面、以什么身份出面、现场边界、客户高层关系、风险修复和会后承接的信息。

创建路径：

```text
memory/customers/[客户名称]/[YYYY-MM].md
```

建议文件：

- `[YYYY-MM-DD]_executive_visit_snapshot.md`：陪访快照。
- `[YYYY-MM-DD]_executive_relationship.md`：客户高层关系卡。
- `[YYYY-MM-DD]_executive_visit_event.md`：高层陪访/升级事件。
- `[YYYY-MM-DD]_risk_boundary.md`：风险、敏感点与承诺边界。
- `[YYYY-MM-DD]_follow_up.md`：会后承接记录。

## Memory 更新规则

- 每次高层陪访、升级汇报或客户高层会面后，必须当天更新“高层陪访事件”和“会后承接记录”。
- 客户高层态度变化、预算/审批变化、竞品阶段变化、交付争议、回款异常、历史承诺风险必须当天更新。
- 领导现场承诺、未承诺但客户可能形成期待的事项，必须写入“承诺边界”。
- 客户内部政治、绕人风险、不能提的话题，必须写入“风险与陪访边界”。
- 与 CRM 冲突时，以 CRM 为准，并在 Memory 中标注“需 AE 核对”。
- 只记录会影响领导判断和会后承接的信息；普通沟通流水放 CRM，不放 Memory。
