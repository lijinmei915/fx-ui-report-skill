# 高层陪访参考

本文件用于判断领导是否应该出面、以什么角色出面、哪些动作由领导做更合适，以及哪些边界不能越过。

## 适合申请领导陪访的场景

| 场景 | 判断标准 | 领导价值 |
| --- | --- | --- |
| 高金额关键商机 | 金额高、预计结单近、客户要求高层背书 | 提升重视度，稳定客户信心 |
| 客户高层参会 | CEO、董事长、业务 VP、CFO、CIO 等会面 | 对等沟通，避免 AE 层级不匹配 |
| 预算/审批卡点 | 项目价值认可但预算、采购、法务或审批迟滞 | 探测真实阻力，推动决策 |
| 竞品深度介入 | 竞品进入 POC、报价、招投标或高层关系 | 稳住关键人，重新定义比较标准 |
| 续约/增购窗口 | 老客户进入续约、扩容或集团化复制窗口 | 强化合作信任，打开新预算 |
| 风险修复 | 交付、回款、承诺、投诉或客户高层不满 | 代表公司态度，修复关系 |

## 不适合申请领导陪访的场景

- 客户需求尚未明确，只是普通初访。
- AE 没有明确希望领导做什么。
- 关键人、预算、竞品、项目价值都不清楚。
- 只是想借领导名义“催客户反馈”。
- 客户当前只需要产品细节、报价细节或实施排期，这些应由 AE、售前、CSM 或实施负责。

## 领导可扮演的角色

| 角色 | 适用场景 | 典型动作 | 不该做 |
| --- | --- | --- | --- |
| 公司背书者 | 客户担心公司实力、长期服务或大项目交付 | 讲公司投入、资源保障、长期合作态度 | 直接承诺不可控交付时间 |
| 业务共创者 | 客户高层关注业务结果、组织管理、经营指标 | 和客户高层讨论业务目标、管理抓手 | 讲产品功能细节 |
| 关系稳定者 | 历史承诺、交付、投诉或竞品冲击导致关系不稳 | 承认问题、表达重视、给出修复机制 | 替 AE 现场背锅或过度让步 |
| 决策探测者 | 预算、审批、采购路径不清晰 | 以高层视角询问决策标准和时间表 | 直接催签或逼问预算 |
| 竞争破局者 | 竞品关系强、报价压制或方案标准被竞品定义 | 重塑比较维度、强调长期价值 | 低层次攻击竞品 |
| 内部协调者 | 需要售前、实施、CSM、产品等资源投入 | 明确我方资源投入和责任机制 | 现场承诺超出授权的资源 |

## 领导陪访目标写法

目标必须具体到“身份 + 动作 + 预期效果”。

```text
建议：以公司业务负责人身份与客户运营 VP 对齐项目价值，确认客户衡量 POC 成功的标准。
不建议：协助 AE 推进项目。
```

可用目标类型：

- 对齐价值：确认客户高层最看重的经营结果。
- 稳定关系：修复历史承诺、交付或服务争议带来的不信任。
- 探测预算：了解预算是否真实存在、是否已进入审批。
- 重塑标准：在竞品对比中把客户关注点从价格拉回长期价值。
- 资源背书：说明我方对交付、服务、产品资源的投入边界。
- 建立高层通道：让客户知道后续关键问题可以通过正式机制升级。

## 领导不能做的事

- 不替 AE 承诺具体报价、折扣、交付日期、定制范围。
- 不绕过客户 Coach 或项目发起人直接找更高层压人。
- 不把客户内部矛盾公开点破。
- 不主动揭短历史承诺或交付问题，除非客户先提或修复关系需要。
- 不做产品演示，不陷入 API、字段、流程等技术细节。
- 不说“我们领导都来了，你们是不是该定了”这类施压表达。

## 风险表达方式

领导简报要诚实，但要给出可执行路径。

| 不推荐 | 推荐 |
| --- | --- |
| 客户预算不清，风险很大 | 预算仍在审批中，建议领导本次探测“预算是否已有归属和审批时间表” |
| 竞品在抢客户 | 竞品已进入深度比较，建议领导把讨论拉回长期服务和集团化复制价值 |
| 上次承诺没兑现 | 上次承诺超期，建议领导表达公司重视，并由 AE 会后给出明确补救计划 |
| Gatekeeper 卡人 | 采购侧有节奏控制，建议领导不要绕过项目发起人，先稳住 Coach |

## 输出边界

- 最高风险不超过 3 条。
- 关键人不超过 5 个，领导必看人物不超过 3 个。
- 陪访目标不超过 3 个。
- 每个目标都必须有会后承接人。
- 如果没有明确领导动作，结论应为“信息不足，暂不建议领导直接出面”。
