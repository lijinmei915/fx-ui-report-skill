# R07 高层陪访简报输出

## 目标

把 R01-R06 的分析结果组装成领导可直接阅读的高层陪访简报。输出必须先给领导必读摘要，再展开证据、风险和目标。

## 触发条件

完成 R01-R06 后始终执行。`--full` 模式或客户存在历史合作、合同、订单、回款、客户健康记录时，再合并 R08。

## 输出顺序

1. 领导必读摘要。
2. 陪访背景。
3. 商机价值。
4. 客户高层与关键人。
5. 领导可介入点。
6. 风险与敏感点。
7. 陪访目标。
8. 会后承接建议。
9. 历史合作、财务与客户健康，仅 `--full` 输出。

## 摘要规则

- 第一屏必须让领导知道：是否建议出面、为什么值得去、最大风险、客户关键高层、领导要做什么、领导不能做什么。
- 摘要只写结论，不堆 CRM 字段。
- 每个关键结论必须标注置信度：确认、推断、未知。
- Memory 不足时，摘要第一屏必须提示“Memory 不足，需 AE 补充高层关系和历史敏感点”。
- 有高风险、竞品 C3/C4、预算/交付/回款/投诉风险时，不能在摘要里弱化。

## 会后承接规则

会后承接建议必须按责任角色组织，至少覆盖：

- AE：客户跟进、材料补齐、客户原话记录。
- 销售经理：内部资源协调、价格/商务边界。
- CSM/实施：交付、服务、客户健康问题。
- 售前：方案、POC、技术边界。
- 陪访领导：是否需要二次高层沟通或内部背书。

## 语言规则

- 使用 CRM、销售管理领域专业术语：商机阶段、预算审批、客户高层、EB/TB/UX、Coach、Gatekeeper、回款、应收、客户健康、续约、增购、流失风险。
- 事实和建议分开写；推断必须标注置信度。
- 领导阅读版要短、清楚、诚实，避免 AE 内部黑话。
- 不输出见客前简报语言：今日目标、P0/P1/P2、确认出发、见客就绪度、今天必须守住。
- 不把行业通用判断写成客户真实痛点。

## 缺口处理

如果数据不足，不要补全结论。使用以下格式：

```markdown
### 陪访缺口
| 缺口 | 影响 | 建议补齐方式 |
| --- | --- | --- |
|  |  | 找 AE / 查 CRM / 问 CSM / 问售前 / 问客户 |
```
