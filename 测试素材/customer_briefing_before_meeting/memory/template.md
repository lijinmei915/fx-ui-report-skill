# 客户见客记忆模板

## 用途

本模板用于沉淀“下次见客前必须知道”的客户事实。它不是完整 CRM 记录，也不是会议纪要全文；只记录会影响见客就绪度、会议目标、风险预警和现场行动的问题。

推荐路径：

```text
memory/customers/[客户名称]/[YYYY-MM].md
```

推荐命名：

- `[YYYY-MM]_meeting_memory.md`：常规会后记忆。
- `[YYYY-MM-DD]_critical_event.md`：关键事件。
- `[YYYY-MM-DD]_relationship_change.md`：关键人关系变化。

## 更新规则

- 每次有效客户互动后，只更新会影响下次见客的信息，不写流水账。
- 承诺、预算、EB、Coach、Gatekeeper、必选条件、竞品阶段变化必须当天更新。
- 信息不确定时，写“未知”或“推断”，不要补成事实。
- 与 CRM 冲突时，以 CRM 为准，并在 memory 中标注“需核对”。
