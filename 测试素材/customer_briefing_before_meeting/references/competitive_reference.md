# 竞品参考

仅在 R04 识别到竞品，或用户要求竞品定位时读取本文件。竞品未知的普通简报不要加载，除非需要生成探查问题。

## 竞品定位矩阵

| 厂商 | 典型客户 | 优势 | 短板 | 纷享销客更适配的场景 |
| --- | --- | --- | --- | --- |
| 销售易 | 100-1000 人中型企业 | 营销模块、用户体验、移动端 | PaaS/定制能力较弱，价格偏高，大客户服务深度有限 | 大型或复杂流程、PaaS 定制、制造/ICT 大客户 |
| 红圈 CRM | 50-300 人外勤销售团队 | 外勤打卡、低价、上线快 | 复杂销售流程弱，BI/集成弱，定制有限 | 200 人以上团队、渠道/流程复杂、需要完整 CRM |
| 神州云动 / CloudCC | 500-5000 人复杂企业 | PaaS 和 ERP 集成能力强 | 实施周期长、复杂度高、用户体验弱 | 更快上线、更好推广使用、PaaS 与易用性的平衡 |
| Dynamics 365 | 跨国企业或微软生态重度客户 | 微软生态、国际品牌 | 价格高、实施复杂、本地适配弱 | 中国本地运营、快速本地实施、非微软重度团队 |
| Salesforce | 跨国企业 | 全球 CRM 生态成熟 | 成本高、本地集成和实施负担重 | 国内企业、成本控制、本地流程和生态适配 |

## R04 差异化代码

| 代码 | 使用条件 |
| --- | --- |
| D1 | 我方有客户关键场景下竞品很难替代的能力 |
| D2 | 我方至少在两个相关维度明显更强 |
| D3 | 能力接近，胜负取决于价格、服务、适配度或关系 |
| D4 | 竞品在客户明确优先事项上有明显优势 |

如果客户真实优先级未知，默认 `D3`，并给出探查问题。

## 场景化竞品话术

用“场景适配”语言，不用“我们更好”。

### 对比销售易

```markdown
如果你们后面会有经销商、审批流、区域差异这些深度定制，纷享的 PaaS 更合适；
如果你们只需要标准化营销流程和更轻的使用体验，销售易也可能省心。
```

### 对比红圈

```markdown
如果核心只是外勤打卡和拜访记录，红圈足够轻；
如果你们超过 200 人，还要统一商机、渠道、审批和报表，纷享更合适。
```

### 对比神州云动

```markdown
如果你们可以接受长周期重实施，神州云动 PaaS 很强；
如果业务部门需要更快上线、更多人愿意用，纷享的实施和体验更稳。
```

### 对比 Dynamics / Salesforce

```markdown
如果你们是全球统一 IT 架构，国际产品有优势；
如果主要是中国团队落地、微信/本地审批/本地服务响应，纷享更贴近。
```

## 竞品阶段线索

| 信号 | 建议阶段 |
| --- | --- |
| 客户问“你们和某某有什么区别” | C1 或 C2 |
| CRM 商机中列出竞品 | C2 |
| 客户看过竞品演示 | C2-C3 |
| 客户正在跑竞品 POC | C3 |
| 客户已经使用竞品 | C4 |

阶段必须有证据。如果来源较弱，置信度标为 `inferred`。

## 探查问题

竞品参与情况不清时，可问：

- “你们现在除了我们，还有没有在看其他方案？”
- “如果最后做选择，您最看重的是价格、上线速度、定制能力，还是现有系统集成？”
- “之前看过的方案里，有哪一点是您觉得必须保留的？”

## 安全规则

- 不攻击竞品。
- 没有当前来源时，不声称市场份额或近期中标率。
- 没有价格证据时，不说“我们更便宜”。
- 所有差异化都必须连接到客户已表达或待验证的具体场景。
