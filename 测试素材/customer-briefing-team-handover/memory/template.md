# 团队交接记忆模板

## 用途

本模板用于沉淀“接手 AE 必须继承”的客户知识。它不是完整会议纪要，也不是 CRM 字段备份；只记录会影响客户交接、关系延续、承诺兑现、风险规避和接手节奏的信息。

推荐路径：

```text
memory/customers/[客户名称]/[YYYY-MM].md
```

推荐命名：

- `[YYYY-MM]_handover_snapshot.md`：交接快照。
- `[YYYY-MM-DD]_critical_event.md`：关键博弈事件。
- `[YYYY-MM-DD]_relationship_card.md`：关键人关系变化。
- `[YYYY-MM-DD]_commitment_ledger.md`：承诺台账。
- `[YYYY-MM-DD]_risk_note.md`：风险与敏感点。

## Memory 更新规则

- 只记录会影响交接的信息，不写流水账。
- 承诺、未兑现事项、关键人态度变化、竞品阶段变化、预算/审批变化、交付争议、回款异常必须当天更新。
- 关系温度必须有行为证据，不要只写“关系好/关系一般”。
- 信息不确定时写“推断”或“未知”，不要补成事实。
- 与 CRM 冲突时，以 CRM 为准，并在 Memory 中标注“需原 AE 核对”。
- 接手人需要口头了解但无法写完整的事项，也要写成“需原 AE 补充问题”。
