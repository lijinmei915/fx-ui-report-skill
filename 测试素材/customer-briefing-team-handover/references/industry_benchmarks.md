# 行业与交接风险参考

本文件用于补充团队交接简报中的行业背景、常见客户风险和接手注意事项。行业基准只能形成“推断”或“待核对事项”，不能替代客户事实。

## 使用边界

- 可以用于解释客户所处行业、采购周期、常见风险和接手关注点。
- 不可以把行业常见问题写成客户已确认事实。
- 不可以用行业基准覆盖 CRM、客户原话、合同、订单或 Memory。
- 引用本文件形成的判断，置信度最高只能是“推断”。

## 行业交接关注点

| 行业 | 常见决策链 | 交接时重点核对 |
| --- | --- | --- |
| ICT/高科技 | CEO/COO、销售负责人、IT、财务 | 系统集成、安全要求、技术评审人、是否有自研替代 |
| 制造业 | 业务负责人、IT、生产/服务负责人、财务 | 经销商/售后流程、ERP/MES 集成、实施窗口、现场服务历史 |
| 消费品 | 销售、市场、渠道、IT、财务 | 渠道政策、终端数据、促销费用、外勤管理、区域负责人关系 |
| 医疗健康/IVD | 业务负责人、服务负责人、IT、合规/财务 | 合规要求、服务工程师管理、经销商数据、医院客户触达边界 |
| 家居建材 | 老板、销售负责人、渠道负责人 | 经销商关系、门店执行、价格秩序、老板个人偏好 |
| 教育培训 | 创始人、运营负责人、销售负责人 | 线索转化、续费、服务满意度、预算敏感度 |

## 采购与接手风险

| 信号 | 可能含义 | 交接写法 |
| --- | --- | --- |
| 预计结单日期临近 | 接手后时间窗口紧，容易丢上下文 | “接手第一周需确认审批/预算状态” |
| 最近 60 天无互动 | 关系可能冷却，原 AE 上下文缺失 | “需原 AE 补充最后一次有效沟通结论” |
| 商机阶段停滞 | 可能有隐性阻力或竞品影响 | “需核对停滞原因，不能只看 CRM 阶段” |
| 主联系人变更 | 决策链可能重洗 | “需确认旧 Coach 是否仍有影响力” |
| 合同临近到期 | 续约风险或增购窗口 | “需核对客户健康、使用效果和 CSM 反馈” |
| 应收金额异常 | 商务/满意度/流程风险 | “不能直接推新单，需先了解应收原因” |

## 常见行业痛点假设

### ICT/高科技

- 销售过程不透明，管理层难以掌握真实进展。
- 系统集成和数据安全是技术评审重点。
- 自研系统或已有工具替代风险较高。

### 制造业

- 经销商管理、售后服务和跨系统集成常影响项目成败。
- 现场流程复杂，实施窗口和业务配合度很关键。
- 老项目可能存在交付体验和服务响应的历史包袱。

### 消费品

- 渠道、终端、促销费用和外勤拜访是高频关注点。
- 区域负责人和总部负责人可能诉求不一致。
- 接手时要核对是否存在区域试点失败或执行阻力。

### 医疗健康/IVD

- 合规、安全、服务过程追踪和经销商数据是常见关注点。
- 服务工程师、医院客户和经销商之间的数据链路复杂。
- 接手时要核对是否有合规红线或客户投诉记录。

## 接手建议生成规则

- 行业假设只能进入“需核对问题”，不能进入“已确认风险”。
- 如果 Memory 与行业基准冲突，以 Memory 和 CRM 事实为准。
- 接手建议要具体到“找谁核对什么”，不要写“持续关注行业变化”。
- 对历史合作客户，优先核对客户健康、续约窗口、未解决问题和应收金额。
