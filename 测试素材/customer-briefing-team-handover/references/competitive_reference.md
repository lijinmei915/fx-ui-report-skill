# 竞品历史参考

本文件用于解释团队交接简报中的竞品历史、当前竞争状态和接手注意事项。竞品参考只能辅助判断，不能替代客户现场证据。

## 使用边界

- 有 CRM、Memory、客户原话或用户输入提到竞品时，读取本文件。
- 没有客户证据时，不主动断言竞品在场。
- 本文件中的强弱项是通用参考，交接简报仍以客户实际评价、报价、POC 和审批反馈为准。
- Handover 场景重点不是写进攻话术，而是让接手人知道“客户为什么比较、历史怎么打、现在谁占优”。

## 竞品阶段

| 阶段 | 名称 | 判断标准 |
| --- | --- | --- |
| C0 | 未知 | 没有竞品信息 |
| C1 | 线索 | 客户提到竞品或同类系统，但没有明确比较动作 |
| C2 | 在场 | 客户明确比较竞品、索要对比或 CRM 有竞品明细 |
| C3 | 深度评估 | 竞品进入 POC、方案、报价、招投标或技术评审 |
| C4 | 已选择 | 客户已签竞品、历史输给竞品或流失原因明确为竞品替换 |

## 交接必须记录的竞品信息

| 信息 | 为什么重要 |
| --- | --- |
| 竞品名称 | 接手人知道对手是谁 |
| 竞品阶段 | 判断竞争压力和项目紧急度 |
| 客户比较原因 | 判断客户真实评估标准 |
| 已知报价 | 判断商务压力和让价边界 |
| 我方历史应对 | 避免接手人重复无效打法 |
| 输赢结果 | 判断客户对我方的历史印象 |
| 关键影响人 | 判断竞品关系在哪个关键人处更强 |

## 主要竞品画像

### 销售易

- 定位：营销服一体化 CRM，偏标准化 SaaS。
- 常见强项：营销模块、界面体验、移动端、品牌认知。
- 常见短板：复杂 PaaS 定制、大型企业复杂流程、深度本地化交付。
- 交接关注：客户是否因“营销功能/品牌/体验”认可销售易，是否已经做过功能对比或报价。

### 红圈营销 / 红圈 CRM

- 定位：移动 CRM 和外勤销售管理。
- 常见强项：外勤打卡、拜访记录、上手快、价格低。
- 常见短板：复杂商机、渠道、报表、集成和定制能力不足。
- 交接关注：客户是否预算敏感，是否只需要轻量外勤管理。

### 神州云动 / CloudCC

- 定位：企业协同和 PaaS 平台，覆盖部分 CRM 场景。
- 常见强项：PaaS、复杂流程、与大型系统集成。
- 常见短板：产品复杂、实施周期长、用户体验和上线速度压力。
- 交接关注：客户是否由 IT 主导、是否强调重度定制或集团级流程。

### Microsoft Dynamics 365

- 定位：全球企业级 CRM。
- 常见强项：Microsoft 生态、全球最佳实践、跨国企业治理。
- 常见短板：本地化、实施成本、周期、国内服务响应。
- 交接关注：客户是否外资背景、是否有集团统一系统要求。

### Salesforce

- 定位：全球 CRM 平台。
- 常见强项：平台生态、成熟度、国际化。
- 常见短板：国内价格、实施成本、本地化和合规适配。
- 交接关注：客户是否已有海外 Salesforce 基础或集团指定。

### 自研系统

- 定位：客户内部 IT 或业务团队自行建设。
- 常见强项：贴合内部流程、数据可控、沉没成本低。
- 常见短板：产品化不足、维护成本、迭代速度、移动端体验、跨部门推广。
- 交接关注：客户内部 IT 话语权、已有系统沉没成本、是否存在“先自研试试”的决策倾向。

## 交接输出规则

- 竞品只要进入 C2 及以上，必须写入交接简报。
- 有竞品报价时，写“已知报价”；没有报价时写“未知”，不要估算。
- 历史输给竞品时，必须写输的原因，以及该原因是否仍影响当前关系。
- 如果竞品只是行业常见玩家但客户未提及，不写入当前竞品历史。
- 如果客户比较自研，要把自研作为竞品处理。
- 接手建议应写“接手人需要核对什么”，而不是直接写攻击竞品的话术。
