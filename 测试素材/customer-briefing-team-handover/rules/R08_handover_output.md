# R08 交接简报输出

## 目标

把 R01-R07 的分析结果组装成接手人可直接阅读的团队交接简报。默认直接输出在聊天正文中，必须先给结论摘要，再展开证据和细节。

## 触发条件

完成 R01-R07 后始终执行。`--full` 模式或客户存在历史合作、合同、订单、回款、客户健康记录时，再合并 R09。

## 输出顺序

1. 接手人必读摘要。
2. 交接背景。
3. 客户画像。
4. 博弈编年。
5. 关系温度与关键人地图。
6. 待兑现承诺。
7. 风险与敏感点。
8. 竞品历史。
9. 接手建议。
10. 历史合作、财务与客户健康，仅 `--full` 输出。

## 摘要规则

- 第一屏必须让接手人知道：当前状态、最大历史包袱、最关键关系、最高风险、第一周重点。
- 摘要只写结论，不堆字段。
- 每个关键结论必须标注置信度：确认、推断、未知。
- Memory 不足时，摘要第一屏必须提示“Memory 不足，需原 AE 补充关键历史”。
- 有高风险、逾期承诺、竞品 C3/C4、回款/交付/投诉风险时，不能在摘要里弱化。

## 接手建议规则

接手建议必须按“第一周可执行”组织，至少包含：

- 第一优先级：先补哪类信息。
- 联系顺序：先找原 AE、CSM、售前、实施、客户 Coach、客户关键人中的谁。
- 不能做什么：哪些话不能说、哪些人不能绕、哪些历史不能假装不知道。
- 需要原 AE 当面交代的问题。
- 下一次客户沟通前必须确认的事实。

## 输出契约

默认输出为聊天正文中的结构化文本。可以使用 Markdown 标题、引用和表格排版，但不要创建 `.md` 文档、附件、文件卡片或本地文件。只有用户明确要求导出、保存、生成文件、`.md`、Word、PDF、HTML、移动端或可分享版本时，才输出对应文件格式。

## 语言规则

- 使用 CRM、销售管理领域专业术语：商机阶段、关键人、Coach、Gatekeeper、EB/TB/UX、承诺、回款、应收、客户健康、续约、增购、流失风险。
- 事实和建议分开写；推断必须标注置信度。
- 不写“关系良好，继续维护”这类空话。
- 不输出见客前简报语言：今日目标、P0/P1/P2、确认出发、见客就绪度、今天必须守住。
- 不使用营销式大词替代客户事实。
- 不把默认输出包装成 `.md` 文件或文档卡片；正文就是最终交付物。

## 缺口处理

如果数据不足，不要补全结论。使用以下格式：

```markdown
### 交接缺口
| 缺口 | 影响 | 建议补齐方式 |
| --- | --- | --- |
|  |  | 找原 AE / 查 CRM / 问 CSM / 问客户 |
```
