# 客户360记忆模板

## 用途

本模板用于沉淀客户长期认知，不是会议纪要全文，也不是 CRM 字段备份。只记录会影响客户画像、博弈时间线、关系温度、决策链、人力地图、项目背景、IT 现状、使用规模、商机全貌、痛点地图、竞品全貌、风险矩阵和开放问题的事实。

推荐路径：

```text
memory/customers/[客户名称]/[YYYY-MM].md
```

推荐命名：

- `[YYYY-MM]_customer360_snapshot.md`：月度客户360快照。
- `[YYYY-MM-DD]_critical_event.md`：关键博弈事件。
- `[YYYY-MM-DD]_relationship_change.md`：关键人关系变化。
- `[YYYY-MM-DD]_competitive_event.md`：竞品事件。
- `[YYYY-MM-DD]_health_risk.md`：客户健康或续约风险。

## 更新规则

- 只记录会改变客户360认知的信息，不写流水账。
- 客户原话、承诺、未兑现事项、竞品阶段、EB/Coach/Gatekeeper 变化必须当天更新。
- CSM 预警、回款异常、流失原因和历史交付争议必须进入 full 模式可读取的记忆。
- 信息不确定时写“推断”或“未知”，不要补成事实。
- 与 CRM 冲突时，以 CRM 为准，并在 memory 中标注“需核对”。
- 同一信息同时影响多个维度时，只写一次事实，再在影响分析中标明影响范围。
