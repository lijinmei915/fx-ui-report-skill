# 竞品参考

本文件用于解释客户360中的竞品格局和常见对比维度。竞品参考只能辅助判断，不能替代客户现场证据。

## 使用边界

- 有 CRM、Memory、客户原话或用户输入提到竞品时，读取本文件。
- 没有客户证据时，不主动断言竞品在场。
- 本文件中的强弱项是通用参考，客户当前项目仍以实际评价、报价、POC 和审批反馈为准。
- 输出客户360时，优先写“客户为什么比较这个竞品”，其次才写通用差异。

## 竞品阶段

| 阶段 | 名称 | 判断标准 |
| --- | --- | --- |
| C0 | 未知 | 没有竞品信息 |
| C1 | 线索 | 客户提到竞品或同类系统，但没有明确比较动作 |
| C2 | 在场 | 客户明确比较竞品、索要对比或 CRM 有竞品明细 |
| C3 | 深度评估 | 竞品进入 POC、方案、报价、招投标或技术评审 |
| C4 | 已选择 | 客户已签竞品、历史输给竞品或流失原因明确为竞品替换 |

## 主要竞品画像

### 销售易

- 定位：营销服一体化 CRM，偏标准化 SaaS。
- 常见强项：营销模块、界面体验、移动端、品牌认知。
- 常见短板：复杂 PaaS 定制、大型企业复杂流程、深度本地化交付。
- 常见客户比较原因：功能相似、品牌熟悉、营销自动化诉求、已有同行案例。

### 红圈营销 / 红圈 CRM

- 定位：移动 CRM 和外勤销售管理。
- 常见强项：外勤打卡、拜访记录、上手快、价格低。
- 常见短板：复杂商机、渠道、报表、集成和定制能力不足。
- 常见客户比较原因：预算敏感、销售外勤管理是核心问题、团队规模较小。

### 神州云动 / CloudCC

- 定位：企业协同和 PaaS 平台，覆盖部分 CRM 场景。
- 常见强项：PaaS、复杂流程、与大型系统集成。
- 常见短板：产品复杂、实施周期长、用户体验和上线速度压力。
- 常见客户比较原因：客户强调重度定制、集团级流程、IT 主导选型。

### Microsoft Dynamics 365

- 定位：全球企业级 CRM。
- 常见强项：Microsoft 生态、全球最佳实践、跨国企业治理。
- 常见短板：本地化、实施成本、周期、国内服务响应。
- 常见客户比较原因：外资背景、Microsoft 全家桶、全球统一系统要求。

### Salesforce

- 定位：全球 CRM 平台。
- 常见强项：平台生态、成熟度、国际化。
- 常见短板：国内价格、实施成本、本地化和合规适配。
- 常见客户比较原因：跨国公司、集团统一、已有海外 Salesforce 基础。

### 自研系统

- 定位：客户内部 IT 或业务团队自行建设。
- 常见强项：贴合内部流程、数据可控、沉没成本低。
- 常见短板：产品化不足、维护成本、迭代速度、移动端体验、跨部门推广。
- 常见客户比较原因：IT 话语权强、已有系统基础、预算受限、数据安全敏感。

## 常见对比维度

| 维度 | 应看客户证据 | 客户360写法 |
| --- | --- | --- |
| 价格 | 报价、采购反馈、压价记录 | “价格压力来自哪家竞品，报价是否已确认” |
| PaaS 定制 | 客户流程复杂度、定制清单、IT 反馈 | “客户是否真的需要复杂定制，还是只是担心适配” |
| 上线速度 | 项目时间表、业务窗口、合同期限 | “上线窗口是否影响选型” |
| 集成能力 | ERP/MES/财务/自研系统接口要求 | “集成是技术门槛还是决策借口” |
| 行业案例 | 客户提到的同行、招标要求 | “同行案例是否是硬性评估项” |
| 服务交付 | 实施周期、服务商质量、CSM 预警 | “客户是否担心买完没人管” |
| 数据安全 | 合规、权限、审计、私有化 | “安全是否为真实红线” |

## 输出规则

- `CompetitiveLinesObj` 有竞品记录时，至少写入 C2。
- 有竞品报价时，写入“已知报价”；没有报价时写“未知”，不要估算。
- 历史输给竞品时，记录原因和是否仍影响当前项目。
- 如果竞品只是行业常见玩家但客户未提及，不写入“当前竞品格局”。
- 如果客户比较自研，要把自研当作竞品，不要忽略。
